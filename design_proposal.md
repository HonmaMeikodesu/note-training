# 视唱练耳与相对音感训练 APP 设计提案 (Design Proposal)

本提案旨在为个人量身定制一款基于 **React Native (Expo)** 技术栈的视唱练耳（Ear Training）与相对音感（Relative Pitch）训练 APP。
通过科学的训练方法、人性化的交互设计以及每日量化的反馈，帮助你在日常琐碎时间中高效提升听音与视唱能力。

---

## 1. 相对音感训练的科学原理与方法

在设计 APP 之前，我们需要明确**相对音感（Relative Pitch）**的训练核心。许多传统的视唱练耳软件偏重于“孤立音程听辨”（例如：播放两个音，让你猜是纯五度还是大三度）。这种方法虽然有用，但对于实际音乐场景（如听歌起谱、即兴演奏、合唱等）转化率较低。

现代音乐认知学更推崇**首调唱名法（Solfège / Functional Pitch Training）**。
- **核心逻辑**：建立“调性感觉（Tonal Center）”。先播放一个和弦进行（如 I-IV-V-I），在大脑中建立稳定的主音（Do）感觉，然后播放一个目标音，训练大脑去感知这个音在当前调性中的**功能/位置**（是稳定的 1、3、5，还是具有强烈倾向性的 7、4，或是过渡性的 2、6）。
- **训练成效**：通过首调功能训练，你可以快速做到“听歌能立刻在脑海中浮现出简谱（1 2 3 4 5 6 7）”，这是最实用的相对音感。

因此，本 APP 的核心设计将围绕**首调功能音感训练**展开，辅以**经典音程训练**和**旋律跟弹训练**。

## 2. APP 核心功能模块设计

为了不让训练变得枯燥，且能提供每天规律训练的动力，我们将 APP 划分为以下几个核心模块：

### 核心功能 1：主面板 & 每日打卡 (Dashboard & Daily Streak)
- **每日任务进度条**：例如每天训练 3 组，每组 10 题，完成即打卡。
- **连续打卡天数 (Streak)**：引入游戏化机制，激发每日训练的动力。
- **历史周/月报表**：展示正确率曲线，直观看到自己的成长。

### 核心功能 2：首调功能音感训练 (Functional Pitch Trainer) —— 【主打功能】
- **训练流程**：
  1. 自动播放当前调性的 **I-IV-V-I 建立和弦（Cadence）**，确立调性。
  2. 播放一个**目标单音**。
  3. 用户在屏幕上选择这是几号音（1, 2, 3, 4, 5, 6, 7）或对应的简谱。
  4. 给出即时反馈：答对亮绿，答错亮红并播放正确音的走向（例如引导音如何解决到主音）。
- **自定义难度设置**：
  - **初级**：仅 C 大调，仅自然音级 (1, 3, 5) —— 建立骨干音感。
  - **中级**：加入 2, 4, 6, 7，随机常用调性（F大调、G大调等）。
  - **高级**：加入变化音/离调音（如 #4, b7 等），甚至小调系统。

### 核心功能 3：音程对比与听辨 (Interval Trainer) —— 【基础巩固】
- **音程对比**：播放两个音程（如 C-E 和 C-F），选择哪个更宽。
- **音程听辨**：播放一个音程（旋律音程或和声音程），选择其类型（如大三度、纯五度）。
- **音程视唱辅助**：给出起始音，屏幕上显示目标音程，用户在脑海中唱出，然后点击“播放”自我核对。

### 核心功能 4：旋律听写与跟弹 (Melody Playback) —— 【进阶训练】
- **训练流程**：
  1. 系统随机生成一段 3~5 个音的短旋律。
  2. 用户在屏幕底部的**虚拟钢琴键盘**上弹奏出听到的旋律。
  3. 系统给出评级（完美、不错、加油）。
- 极大地锻炼“听音-手指反射”能力，非常适合乐手和键盘爱好者。

### 核心功能 5：多功能交互键盘 (Interactive Keyboard & Sandbox)
- APP 底部或独立页面常驻一个高品质的虚拟钢琴键盘。
- 支持标记唱名（固定音高名 C D E 或首调简谱 1 2 3）。
- 随时作为你视唱时的“定音器”使用。

## 3. 技术栈选型与架构方案

基于 **React Native + Expo** 的轻量高效要求，推荐使用以下技术组合：

| 模块 | 推荐库 | 作用与优势 |
| :--- | :--- | :--- |
| **基础框架** | `Expo SDK (TypeScript)` | 零配置、快速开发、易于真机调试、跨平台。 |
| **音频引擎** | `expo-av` | 用于播放高清钢琴采样（WAV/MP3），确保发音零延迟或低延迟。 |
| **数据持久化** | `@react-native-async-storage/async-storage` | 存储每日打卡记录、关卡进度、用户配置，无需后端，全本地运行。 |
| **路由导航** | `react-navigation` / `expo-router` | 实现主页、训练页、统计页、设置页的丝滑切换。 |
| **UI 组件库** | `react-native-svg` + `NativeWind` (or `styled-components`) | 绘制极简、现代、带微动效的扁平化训练界面。 |
| **动画效果** | `react-native-reanimated` | 用于答题正确/错误时的卡片翻转、进度条流动等解压动效。 |

## 5. 项目分阶段实施计划 (Todo List)

为了稳步推进，我们将开发分为三个阶段：

### 阶段一：原型设计与技术验证 (MVP)
1. 确定音频播放方案，在 Expo 环境中实现多音轨、低延迟单音播放。
2. 制作核心训练算法（随机生成音符、音程，计算正确率）。
3. 制作最基础的首调听音页面。

### 阶段二：功能拓展与游戏化
1. 引入打卡机制与本地存储，实现连续天数、每日进度计算。
2. 增加音程听辨和旋律跟弹模式。
3. 增加带琴键提示的虚拟键盘。

### 阶段三：视觉打磨与体验优化
1. 设计柔和清新的 UI 界面，加入解压反馈动效。
2. 优化音频加载速度，提供音高微调、速度控制、乐器切换（钢琴/合成器）等高级设置。
3. 全面测试并打包。
